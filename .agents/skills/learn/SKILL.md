# learn — 영역의 LEARNED_CAUTIONS.md에 학습된 주의사항 누적

## 입력 형태

| 호출 | 동작 |
|------|------|
| `/learn <메모>` (Codex는 `$learn <메모>`) | 사용자가 적은 메모를 그대로 누적 |
| `/learn` (인자 없음) | 최근 대화에서 잘못된 내용을 한 줄로 추출해 누적 |

## 절차

1. **메모 확정**
   - 인자가 있으면: 인자 텍스트를 그대로 메모로 사용 (윤색/요약 금지)
   - 인자가 없으면: 최근 대화 맥락에서 잘못된 내용을 한 줄 메모로 추출
     - 단서: 사용자의 정정 발화, 잘못된 가정으로 인한 실패, 코드만 봐선 알 수 없던 함정이 드러난 순간
   - 사용자 승인은 5단계 통합 게이트에서 일괄 처리한다 (여기서는 메모 확정만)

2. **현재 작업 영역 추론**
   - root map 파일(`CLAUDE.md` 또는 `AGENTS.md`)의 "영역별 가이드" 섹션을 읽어 후보 영역 목록 확보
   - 다음 우선순위로 영역 단서를 본다:
     1. 대화에 명시된 경로 (예: "apps/frontend/Login.tsx 수정")
     2. 최근 Edit/Write로 수정한 파일의 위치
     3. 키워드 매칭 (스택/도메인 용어, 예: "API" → backend)
   - 위 단서가 둘 이상 영역을 가리키거나 0개일 때 = 모호. 모호하면 사용자에게 "어느 영역에 누적할까요?" 라고 물어 확정

3. **대상 파일 결정 — LEARNED_CAUTIONS.md만 갱신**
   - 영역 폴더의 `LEARNED_CAUTIONS.md`가 누적 대상이다. 본문 가이드(`AGENTS.md` / `CLAUDE.md`)는 절대 건드리지 않는다.
   - 파일이 없으면 사용자에게 알리고 중단 (`init` 스킬이 placeholder를 생성했어야 함 — 누락 시 템플릿이 깨졌다는 신호)
   - both 모드와 무관하게 영역당 `LEARNED_CAUTIONS.md`는 하나만 존재. import는 본문 가이드의 8번 섹션에서 처리됨.

4. **중복 체크**
   - 대상 `LEARNED_CAUTIONS.md` 본문을 읽어 동일/유사 내용이 이미 있는지 확인
   - 있으면 사용자에게 알리고 추가 여부 결정

5. **누적 직전 통합 확인 (필수 게이트)**
   - 다음 세 가지를 한 번에 보여주고 승인을 받는다. 인자 유무·영역 모호성과 무관하게 항상 거친다.
     - 추가할 메모 (1단계 결과)
     - 대상 파일 경로 (3단계 결과 — 항상 `{area}/LEARNED_CAUTIONS.md` 형태)
     - 영역 판단 근거 (2단계에서 어떤 단서를 어떻게 해석했는지 한 줄)
   - 사용자가 거절하면 중단. 보정 요청이면 해당 단계로 돌아가 다시 진행.

6. **반영**
   - 형식: `- (YYYY-MM-DD) <메모>` (오늘 날짜 자동 삽입)
   - `LEARNED_CAUTIONS.md` 하단(분할 안내 주석 위)에 추가
   - 파일이 `_(아직 없음)_` placeholder 상태이면 그 줄을 첫 항목으로 교체
   - `LEARNED_CAUTIONS.md` 파일 자체가 없으면 사용자에게 알리고 중단 (init 누락 신호. 임의 생성 금지)

7. **마무리**
   - 누적된 파일 경로 + 추가된 줄을 사용자에게 보여주고 종료
   - 커밋은 자동으로 하지 않는다 (사용자가 직접)

## 원칙

- **본문 가이드는 절대 건드리지 않는다**: `AGENTS.md` / `CLAUDE.md`는 8번 LEARNED CAUTIONS 섹션이 이미 `@./LEARNED_CAUTIONS.md`를 참조하도록 init이 구성해 두었다. learn은 참조 대상 파일만 갱신한다.
- **인자 있는 호출은 메모 윤색 금지**: 사용자가 적은 표현 그대로 누적. 자기 표현이 나중에 검색/이해할 때 가장 정확함.
- **누적 직전 통합 확인은 항상 거친다**: 인자 유무·영역 명확성과 무관하게 "누적 직전 통합 확인" 게이트(절차 5단계)는 생략하지 않는다. 약속(금지)-동작 일관성을 위해 필수.
- **영역이 모호하면 무조건 묻는다**: 잘못된 영역 누적이 가장 큰 실패 모드.

## 금지

- **명시 호출 없이 자동 발동 금지** — 사용자가 `/learn` (또는 `$learn`)을 직접 입력하지 않은 상황에서 "이거 잘못한 것 같으니 누적할까요?" 류로 선제 발동하지 않는다. 이 스킬은 사용자 트리거 전용이다.
- **본문 가이드(`AGENTS.md` / `CLAUDE.md`) 수정 금지** — learn은 `LEARNED_CAUTIONS.md`만 갱신한다.
- **LEARNED_CAUTIONS.md 자동 생성 금지** — placeholder가 없으면 init이 누락된 것이므로 중단하고 사용자에게 알린다.
- 사용자 사전 확인 없이 파일 수정
- 자동 커밋
- 영역이 불명확한데 임의로 추정해서 추가
