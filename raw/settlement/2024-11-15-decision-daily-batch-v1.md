# 결정: 정산 자동 배치 v1 — 일배치(D+1) 집계 (정산팀 1세대)

- 일시: 2024-11-15
- 결정자: 한지우(정산팀 리드), 오세훈(정산 백엔드)

## 배경
가맹점이 20곳을 넘으면서 전날 거래를 가맹점별로 손으로 더해 정산하는 게 한계. 한지우가 매일 엑셀로 거래 내역 받아서 가맹점별로 합산 → 수수료 빼고 → 이체하는데, 건수 늘어나니 합산하다 실수 나고 시간도 너무 걸림. 자동으로 전날 거래를 가맹점별로 집계해주는 게 필요.

## 검토한 대안
1. 실시간 정산: 거래 들어올 때마다 바로 정산 금액 계산. → 환불·취소 정정이 들어오면 이미 계산한 금액을 다시 까야 해서 복잡함. 가맹점별 누적도 꼬임. 탈락.
2. **일배치 정산 (채택)**: 매일 새벽 전날(D) 거래를 모아 다음날(D+1) 가맹점별로 집계.
3. 주배치 정산: 일주일치 모아서 한 번에. → 정산이 너무 늦다고 가맹점에서 불만 들어옴(돈 언제 들어오냐고). 탈락.

## 결정
- **매일 새벽 04:00(KST)에 전날 거래를 가맹점별로 집계하는 배치를 돌린다. D+1 정산(전날 거래를 다음날 정산).**
- 배치가 하는 일: 전날 거래 모으기 → 가맹점별 합계 → 수수료 차감 → 가맹점별 정산 금액 리포트 출력.
- **송금은 배치가 안 한다.** 배치는 합계만 내고, 한지우가 리포트 검토 후 직접 수기 이체. 자동 송금은 아직 무서워서 보류(잘못 쏘면 돈 회수가 안 됨).
- 일단 크론 한 줄로 돌리고 결과 잘 나오는지 한동안 지켜본다. 수기 엑셀에서 막 벗어난 1세대 자동화라, 락이나 멱등 가드 같은 건 안 넣고 단순하게 간다. 사실상 수동 트리거에 가까운 조잡한 버전.

## 근거
- 손으로 합산하던 걸 배치가 대신해주니 사람 실수가 줄고 정산 금액 정확도가 올라감.
- 집계랑 송금을 분리하면, 자동화는 안전한 집계부터 시작하고 돈 나가는 부분은 사람이 한 번 더 확인할 수 있음.

## 리스크
- 배치가 새벽에 죽으면 그날 정산이 통째로 누락됨. 한지우가 아침에 리포트 안 나온 거 보고 알아챌 수밖에 없음(알람 아직 없음).
- 죽어서 다시 돌리면 같은 날 거래가 두 번 집계될 수 있음. 멱등 가드가 없어서 중복 집계를 막을 장치가 아예 없음 — 일단 한지우가 이체 전에 금액 눈으로 확인하는 걸로 커버. (추후 보완 필요)
- 동시에 두 번 돌거나 워커가 겹쳐도 막을 락이 없음. 지금은 사람이 한 번만 돌리는 전제라 그냥 둠.
