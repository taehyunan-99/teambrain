**10:02 정유진**: 📢 [전사공지] Redis 캐시/세션 인프라 오픈했습니다 🙏
**10:02 정유진**: 그동안 팀별로 따로 띄워쓰던 거 정리해서 공용 Redis 클러스터 올렸어요. 캐시랑 세션 저장 용도로 쓰시면 됩니다.

**10:03 정유진**: 접근 정보 정리해둘게요
- 내부 엔드포인트: `redis.internal.nimbuspay.io:6379`
- 네임스페이스는 팀별 prefix 권장 (예: `pay:`, `settle:`, `infra:`)
- 접근은 VPC 내부에서만 가능, 외부 노출 X
- 비밀번호/접속 토큰은 vault `infra/redis/common` 에 넣어놨어요. 권한 필요하면 DM 주세요

**10:05 정유진**: 사용 시 몇 가지 꼭 부탁드릴 게 있어요
1. **모든 키에 TTL 반드시 설정해주세요** 🙏 무한정 사는 키 쌓이면 메모리 터집니다
2. 현재 maxmemory 정책이 `noeviction` 이에요. 즉 메모리 차면 알아서 안 지워지고 **쓰기가 그냥 실패**합니다. 그러니 TTL은 선택이 아니라 필수라고 봐주세요
3. 대용량 키(큰 값/대량 키 발행 예상되는 워크로드)는 사용 전에 인프라팀이랑 한번 상의해주세요. 용량 산정해야 해서요

**10:06 정유진**: noeviction 정책은 일단 안전하게 가려고 그렇게 잡은 거고, 트래픽 패턴 보면서 추후 조정할 수 있어요. 근데 그 전까지는 TTL 관리가 진짜 중요합니다

**10:07 정유진**: 결제팀 정산팀 다들 세션이나 멱등성 같은 데 쓸 일 있을 것 같아서 공유드려요. 도입 검토하실 때 미리 말씀 주시면 같이 봐드릴게요 👍
