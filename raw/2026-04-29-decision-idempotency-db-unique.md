# 결정: 멱등성 키 — DB 유니크 제약 2중화 (3/11 결정 수정)

- 일시: 2026-04-29
- 결정자: 김도현, 박서연
- 관련: INC-204 장애 회고, 2026-03-11 멱등성 결정

## 무엇을 바꾸나
2026-03-11에 "멱등성 = Redis 단독"으로 결정했으나, INC-204(중복결제 87건)로 Redis 단일 의존의 위험이 드러났다. **결정을 수정한다.**

## 새 결정
- 멱등성 저장은 **2계층**:
  1. Redis 캐시(`idem:{key}`, TTL 24h) — 빠른 경로, 응답 재생용.
  2. **`payment_idempotency` 테이블에 `idempotency_key` UNIQUE 제약** — 최종 방어선.
- 결제 INSERT가 유니크 위반(23505)이면 = 중복 → 기존 결제 응답 반환.
- Redis 조회 실패 시 **fail-closed**(거부 후 클라 재시도). 더 이상 통과시키지 않음.

## 3/11 결정 대비 무엇이 유지/변경되나
- 유지: 클라이언트가 `Idempotency-Key` 헤더로 UUID를 보내는 방식. (이건 옳았음)
- 변경: 저장소가 Redis 단독 → Redis + DB 유니크. 타임아웃 정책 fail-open → fail-closed.

## 근거
- 캐시는 휘발성·장애 가능. 돈 문제의 최종 일관성은 DB 트랜잭션 + 유니크 제약이 보장해야 한다.
- INC-204 교훈의 직접 반영.
