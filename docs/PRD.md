# PRD — llmwiki

> Product Requirements Document · 작성일 2026-06-12 · 상태: 설계 확정

## 1. 배경 & 문제

- 흩어진 노트/문서가 연결되지 않아, 다시 찾고 다시 정리하는 비용이 큼.
- 전임 [second-brain](../../second-brain-project/) 은 임베딩·벡터·그래프·4축검색으로 과설계되어 중단됨 (코드 1341줄·문서 5500줄, 라이브 함정 누적).
- LLM이 raw 입력을 읽어 **"스스로 정리·교차참조·최신화"** 하는 wiki가 필요.

## 2. 목표

raw 노트를 던지면 LLM이 `[[wikilink]]`로 연결된 개념 wiki를 **점진적으로 빌드/유지**한다. 사람은 원자재만 던지고, 구조화(개념추출·링크·요약·최신화)는 LLM이 한다 (Andrej Karpathy의 **LLM Wiki** 개념).

핵심 가치: **"정리의 부담을 LLM에게, 사람은 입력과 사고에 집중."**

## 3. 사용자 & 핵심 시나리오

- **주 사용자:** 본인 (개인 지식 관리, 추후 팀 확장 여지).
- **시나리오 1 (덤프→정리):** inbox에 메모를 던지면 raw로 승격 후 개념 아티클 생성.
- **시나리오 2 (점진 성장):** 새 raw를 추가하면 관련 개념 아티클이 머지·갱신되고 새 링크 연결.
- **시나리오 3 (질의):** Claude Code가 wiki를 직접 읽어 "X에 대해 정리해줘"에 답 (임베딩 없이).

## 4. 기능 요구사항

### 필수 (MVP)
1. **지식 입력:** Obsidian vault `raw/`에 수동 작성 (자유 형식, 마찰 0).
2. **wiki 빌드:** raw → 개념 추출 → 개념 아티클 생성/머지 → `[[wikilink]]` 연결.
3. **변경 감지:** 바뀐 raw만 재컴파일 (멱등). source_hash 대조.
4. **출처 추적:** 모든 아티클이 자신을 만든 raw를 `sources`로 기록.
5. **index.md:** 전체 개념 카탈로그 (진입점 + 중복 판단 인덱스).
6. **조용한 실패 금지:** 실패/스킵은 `_log.md` 기록.
7. **손수정 보호:** 사람이 고친 아티클은 빌드가 덮어쓰지 않음.

### 비기능
- **비용/의존 최소:** 외부 API·DB·임베딩 없음. Claude Code + git만.
- **가역성:** 모든 변경은 working tree, 커밋은 사용자 결정. 복구 = git.
- **멱등성:** 안 바뀐 raw는 재방문 안 함.

## 5. 비목표 (현재 범위 제외)

- 임베딩 / 벡터DB / FTS5 / 타입드 그래프 (second-brain 방식 — **의도적 폐기**).
- 사람 review 승인 루프 (자동 적용 + git으로 대체).
- 자동 수집 (Slack/Notion 연동), 웹 UI, 멀티유저 — 추후.
- Hermes Agent 등 범용 자율 에이전트 도입 — wiki 빌드에 불필요 ([ADR-002](ADR.md)).

## 6. 성공 기준

- raw 3~5개 던지면 → wikilink로 연결된 개념 아티클 + index.md 생성.
- 재빌드 시 변경 없으면 스킵 (멱등).
- 손수정한 아티클이 빌드 후에도 보존됨.
- dangling link/실패가 `_log.md`에 남아 추적 가능.

## 7. 기술 스택 (요약)

Claude Code 스킬 (`.claude/skills/*/SKILL.md`) · Obsidian 마크다운 vault · git. 코드 ≈ 0줄.
